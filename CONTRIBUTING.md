# Contributing and Codex Workflow

1. Select one approved task and one milestone.
2. Work in an isolated branch/worktree.
3. Read root and scoped `codex.md` files.
4. Update behavior/data contracts before or with implementation.
5. Keep the diff coherent; do not mix cleanup or unrelated tuning.
6. Run the task's build, tests, replay, persistence, performance, and documentation checks.
7. Package evidence.
8. Request independent adversarial review.
9. Obtain creator-visible acceptance.
10. Merge only through the protected process and update status/release notes.

High-risk changes may not be planned, implemented, verified, and approved by one uninterrupted role/session.
