# Story Authority

## Universal rule
Narrative consumes structured evidence. It may not become a second source of world truth.

## Version 1.0
Version 1.0 requires factual event highlighting, notifications, and short summaries. It does not require a full Story Director or multiple storyteller profiles.

Every candidate names the simulation system that publishes it and the domain system that alone may resolve it. Story code may select or frame an eligible candidate; it never owns the resulting authoritative mutation.

## Later early-modern depth
When a Story Director is activated, it may rank eligible incident candidates, adjust timing within declared bounds, suppress repetition, protect configured recovery windows, prefer connected event chains, choose notification prominence, and frame factual chronicle output.

It may never bypass preconditions, create resources/actors/knowledge, predetermine battle or political outcomes, ignore logistics/law/geography, target because the camera watches, violate deterministic replay, or exceed world/profile limits.

## Verification
Pure Simulation remains the reference. Every story selection delegates resolution to the owning domain and emits a trace. No story system is implemented early merely to satisfy the Version 1.5 architecture horizon.

Observed, unobserved, headless, and different-camera runs must produce identical authoritative checkpoints for the same seed and commands.
