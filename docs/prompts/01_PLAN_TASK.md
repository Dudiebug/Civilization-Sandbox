# Plan a Task

```
Act as the planner. Do not edit files.
Plan [TASK-ID]. Read its contract, blueprint references, scoped codex.md files, relevant specs, current ADRs, tests, and prior evidence.
Produce small ordered milestones. For each: exact files/boundaries, behavior, tests, performance/persistence impact, rollback, and stop condition.
List only genuinely blocking questions. End with a short creator decision: APPROVE PLAN or CHANGES.
```
