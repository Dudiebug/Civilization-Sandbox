# Cognitive-Access Rules

These rules reduce working-memory load and decision fatigue.

## Codex must
- Put the decision or result first.
- Present one next action, not ten alternatives.
- Use sections shorter than one screen when possible.
- Use exact filenames and commands.
- Separate **must decide now** from **can defer**.
- Repeat the active task ID and current status in every handoff.
- Stop after each milestone and wait for approval.
- Summarize failures in plain language before technical detail.
- Preserve all context in repository files so the creator does not need to remember chat history.

## Codex must not
- Ask compound questions.
- replace a requested answer with a large lecture;
- mark work complete because it looks plausible;
- hide uncertainty or silently choose product direction;
- produce a huge implementation diff without approved milestones;
- require the creator to compare raw logs manually.

## Creator response options
Use simple responses: `APPROVE`, `CHANGES`, `STOP`, or `ROLL BACK`.
