# Release Expansion Plan

Do not schedule a later release until the current release is stable and its creator gate is accepted. Each release receives a bounded scope brief, task contracts, benchmarks, migration plan, and reversal trigger.

## Early-modern completion ladder
- [ ] **1.1 Living People - working theme:** candidate depth in people, households, traits, health, skills, social ties, speech, and life history.
- [ ] **1.2 Roads and Markets - working theme:** candidate depth in resources, production, ownership, markets, roads, bridges, shipments, and settlement form.
- [ ] **1.3 States and Beliefs - working theme:** candidate depth in government, legitimacy, factions, religion, culture, policy, and capability divergence.
- [ ] **1.4 War and World - working theme:** candidate depth in diplomacy, treaties, warfare, occupation, ecosystems, disasters, extinction/recovery, powers, and history products.
- [ ] **1.5 Complete Early Modern:** satisfy `VERSION_1_5_PRODUCTION.md` and the original Blueprint v2.0 complete early-modern acceptance target.

These are working allocation themes, not commitments. Exact content is chosen after the prior release using player feedback, risk, architecture maturity, and remaining Version 1.5 gaps.

## Later possibility spaces
- [ ] **2.0 Industrial transition:** steam, machine tools, factories, rail, coal systems, industrial labor, dense pollution, mass education, and larger cities.
- [ ] **3.0 Electrification:** power grids, telecommunications, precision manufacture, utilities, large services, and multicenter cities.
- [ ] **4.0 Mechanization:** automobiles, fuel networks, aircraft, mechanized armies, faster logistics, and new tactical abstractions.
- [ ] **5.0 Atomic/information:** radiation, strategic weapons, missiles, computing, electronic communication, surveillance, and deterrence.
- [ ] **6.0+ Divergent futures:** automation, biotechnology, alternate energy, and social-development paths supported by later research.
- [ ] **Distant expansion:** orbital topology, celestial resources, interplanetary logistics, scale transitions, and multi-planet civilization.

Software versions expand what is possible; they never force every civilization into one global age state.
