# Milestone 0.9 - Release Candidate

**Status:** Planned; task contracts intentionally deferred until Milestone 0.8 passes.
**Detailed program context:** `PRE_1_0_ROADMAP.md`, Section 13.

## Purpose
Turn the integrated god sandbox into a stable, understandable, accessible, recoverable, and supportable Version 1.0 release.

## Entry conditions
- Milestone 0.8 passes all identity guardrails relevant to the implemented game.
- Implemented systems have explicit owners, save semantics, tests, and known limitations.
- D10 is decided from integrated benchmarks and external playtest evidence.

## Player-visible result
A new player can create a world, understand the main causal loop, observe distinct societies, inspect and influence them, experience crisis/conflict/recovery, save, return, and continue without developer assistance.

## Workstreams
1. Final release-scope manifest and content freeze, including exact supported scale and explicit exclusions.
2. World creation, onboarding, notifications, tracking, time controls, explanations, and intervention flow.
3. Accessibility and input review over every core task.
4. Release-critical visual consistency, LOD, occlusion, effects, and placeholder replacement.
5. Save migrations, interrupted-write recovery, corruption diagnostics, backup restore, and support export.
6. Integrated replay, multi-seed, accelerated-run, wall-soak, save-growth, routing, memory, trace, and presentation evidence at the approved envelope.
7. External new-player playtest, independent adversarial review, regression closure, and risk disposition.
8. Packaging, licenses/attributions, build manifest, release notes, known limitations, rollback, and support procedure.

## Integrated gate scenario
From a clean user environment, create and play a world through founding, settlement growth, divergence, connection, crisis/conflict/recovery, each player-control class, inspection, save/exit/restore, and continued simulation. The approved long-run and failure-recovery suites pass without developer intervention.

## Required gate evidence
- approved final scope manifest and decision references;
- onboarding/usability and accessibility results;
- deterministic replay, seed-suite, accelerated-run, soak, performance, save-growth, migration, interruption, corruption, and backup reports;
- clean package/install validation and build manifest;
- external playtest findings and regression ledger;
- independent review and creator release decision.

## Decisions opened
- D10 exact supported scale, stress limits, save horizon, test/soak durations, reference/minimum hardware, platform/distribution target, content lock, and commercial/brand choices genuinely needed for distribution.

## Decisions intentionally left open
Version 1.1-1.5 feature allocation, industrial/later ages, and every nonessential system outside the accepted Version 1.0 loop.

## Stop, hold, or scope-reduction triggers
- Save corruption/recovery or migration is not supportable.
- Core outcomes remain unreadable to new players.
- The accepted release envelope is unstable under soak or accelerated simulation.
- Critical architecture/authority/identity defects remain open.
- Release depends on completing Version 1.5-only breadth.

## Version 1.0 handoff
A passing 0.9 build may be tagged Version 1.0. A post-release evidence review then opens D11 and selects one bounded early-modern depth slice; working 1.1-1.4 themes remain nonbinding until that review.
