# Prompt - Build 10: Living People Version 1.1

Deepen the released Founding Worlds simulation so named people feel like persistent lives rather than interchangeable workers. Preserve all accepted Version 1.0 behavior, saves, causal history, determinism, and player authority.

Player outcome: the player can follow people across childhood or dependency, household change, learning, work, health, relationships, beliefs, possessions, inheritance, speech, migration, and death, and can explain how earlier experiences shaped later choices.

First inspect the current build, Blueprint, scope matrix, measurements, and player feedback. Propose 3-8 independently playable slices and implement only the first approved slice at a time. Favor end-to-end additions such as dependents and caregiving, learning and aptitude growth, active traits and attitudes, durable social ties, contextual speech, possessions/inheritance, or life-history inspection. Build shared infrastructure only when the current slice needs it.

Every slice must update simulation state, presentation, inspection/explanation, persistence, tests, and migration together. Keep ordinary citizens autonomous. Do not use live LLM citizens, a giant global brain, fabricated off-screen outcomes, or presentation-driven authority.

Acceptance: representative named lives remain coherent across save/load and accelerated time; relationships, learning, health, beliefs, and possessions change for inspectable causes; population-scale cost is measured; old worlds migrate safely; and the Unity build remains playable after every accepted slice.

Do not silently cut the remaining Version 1.1 scope. Record deferred depth in the scope matrix and stop for creator decisions when behavior or content breadth materially changes the vision.
