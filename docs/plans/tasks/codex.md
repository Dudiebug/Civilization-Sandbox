# Playable Build Prompt Instructions

- Each `BUILD-*.md` file is a direct creator-to-Codex prompt, not a one-shot demand to implement an entire release in one change.
- Start only the next selected build. Inspect the current project before proposing slices.
- Preserve the prompt's player outcome, acceptance conditions, deferrals, and the repository's Blueprint authority.
- Prefer a visible end-to-end capability over isolated infrastructure. Add supporting structure only when the active slice needs it.
- Stop and ask the creator when a choice would materially alter the game vision, content breadth, authority model, or release outcome.
