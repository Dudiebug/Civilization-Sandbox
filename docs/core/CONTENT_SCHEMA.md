# Content Schema Rules

- Namespaced stable content IDs are immutable and never reused.
- Every definition has schema version, localization keys, explicit units, tags, compatibility metadata, and validator rules.
- Runtime references IDs/descriptors, not editor object names or direct asset paths.
- Removed fields/IDs remain reserved or migrate explicitly.
- Data may select approved evaluators/curves; it may not inject arbitrary executable code.
- Every capability has prerequisites, blockers, visible effects, and scenarios.
- Every building has footprint/entrance, construction/damage/LOD, and descriptor mapping.
- Every policy has authority, administrative cost, enforcement, and expected transaction/service effects.
- Every speech line has context, knowledge safety, grammar/localization, provenance, cooldown, and repetition family.
- Future-era content remains excluded from active Version 1.0 catalogs.
