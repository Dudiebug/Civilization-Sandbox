# Testing and Evidence

## Test layers
1. Pure unit tests for deterministic primitives and validation.
2. Scenario fixtures for behavior contracts.
3. Property tests for invariants.
4. Metamorphic tests for directional relationships.
5. Multi-seed statistical tests for controlled diversity.
6. Save/load and migration fixtures.
7. Canonical replay and state-diff tests.
8. Headless performance benchmarks and long soaks.
9. Presentation smoke and visual-regression evidence.
10. Adversarial architecture review and creator acceptance.

## Evidence pack
Every consequential task records commands, environment, build result, test result, replay result, save impact, performance result, changed contracts/docs, known limitations, independent review, and creator acceptance.

## Completion rule
Compiling is not completion. A visually plausible result is not completion. A passing happy-path test alone is not completion.
