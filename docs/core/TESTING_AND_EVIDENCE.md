# Testing and Evidence

## Test layers
1. Pure unit tests for deterministic primitives and validation.
2. Scenario fixtures for behavior contracts.
3. Property tests for critical invariants.
4. Metamorphic tests for important directional relationships.
5. Multi-seed statistical tests where controlled diversity is part of acceptance.
6. Save/load and migration fixtures.
7. Canonical replay and state-diff tests.
8. Headless performance benchmarks and milestone-sized soaks.
9. Presentation smoke and visual-regression evidence.
10. Adversarial architecture review and creator acceptance.

## Proportionality rule
Not every task needs every layer. The active task contract states which evidence is required. A field may be N/A only with a written reason. Lean scope reduces system breadth, not evidence honesty.

## Evidence pack
Every consequential task records scope and decision record, commands, environment, build result, test result, replay result, save impact, performance result, changed contracts/docs, known limitations, independent review, and creator acceptance.

## Completion rule
Compiling is not completion. A visually plausible result is not completion. A passing happy-path test alone is not completion. A future-release capability is not a hidden prerequisite unless the active release plan says so.
