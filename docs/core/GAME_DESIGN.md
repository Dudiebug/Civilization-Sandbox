# Game Design Contract

## Core loop
Observe → detect change → inspect → explain → choose a stance → watch implementation → compare consequences.

## Player agency
- **Observe and inspect:** search, overlays, traces, history, bookmarks.
- **Binding Order:** obligates the correct decision layer; prerequisites and implementation remain simulated.
- **Indirect intervention:** changes physical, biological, social, or informational conditions; ordinary systems propagate effects.
- **Force Outcome:** bypasses feasibility but must reconcile invariants and permanently record the intervention.

## First-playable baseline
- Standard world: approximately 8 km × 8 km, four founding groups of 40 people.
- Large benchmark: approximately 16 km × 16 km and 10,000 persistent people.
- 18 commodity/resource classes, 16 building archetypes, 16 occupations, six institution types, four government presets, 12 cultural tendencies, six technology domains, 12 diplomatic actions, 24 player powers, and 13 main UI workspaces.

## Content rule
Thin but real implementations must form one vertical causal loop. A disconnected collection of technical demos is not a first playable.
