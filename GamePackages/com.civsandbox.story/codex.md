# com.civsandbox.story

## Owns
Incident candidate catalog, Story Director selection/pacing/profiles, factual chronicles, timelines, historical maps, template grammar, provenance, and repetition controls.

## Forbidden
No fabrication of outcomes/resources/actors/knowledge, no live authoritative prose generation, no camera favoritism.

## Dependency rule
Depends on history and immutable simulation read models; sends only validated candidate commands to owning domain systems.

## Change requirements
- Read the active task and relevant behavior/data contracts.
- Preserve stable IDs, determinism, save compatibility, and explicit units.
- Add positive, negative, boundary, replay/save, and performance evidence appropriate to the change.
- Do not expose mutable internal collections across package boundaries.
- Update package documentation and architecture tests with any public API or dependency change.
